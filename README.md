# Подсистема конфигурации и отладки постамата "KSK Postamat"

![Build Status](https://img.shields.io/badge/build-passing-brightgreen.svg) ![.NET Version](https://img.shields.io/badge/.NET-9.0-blue.svg) ![Company](https://img.shields.io/badge/Company-ЧТУП%20"Руд%20Буд"-orange.svg)

Данный проект разработан в рамках производственной практики (июнь-июль 2026 г.). Система предназначена для настройки конфигурации ячеек постамата, эмуляции взаимодействия с учётной системой 1С и отладки работы контроллера оборудования.

## 🚀 Основные возможности

- **Валидация конфигурации:** Автоматическая проверка JSON-файлов на уникальность ID ячеек, корректность привязки заказов и логическую целостность данных с использованием `FluentValidation`.
- **Реалистичный интерфейс:** Графический терминал на базе `WebView2`, имитирующий физическое расположение ячеек разного размера (малые, средние, большие).
- **Эмуляция оборудования (Mocking):** Полноценная отладка бизнес-процессов (поиск заказа -> оплата -> открытие ячейки) без подключения реального контроллера и системы 1С.
- **Промышленное логирование:** Использование `Serilog` с настроенной ротацией файлов и выводом в реальном времени в консоль и GUI.
- **Unit-тестирование:** Покрытие критических узлов системы (валидаторы, бизнес-логика сервисов) тестами на базе `xUnit`.

## 🛠 Технологический стек

- **Backend:** C# / .NET 9, ASP.NET Core Minimal API.
- **Core:** Модели данных, интерфейсы, FluentValidation.
- **Terminal (GUI):** WinForms, Microsoft Edge WebView2, HTML5, CSS Grid (12-column layout), jQuery.
- **Logging:** Serilog (File sink, Rolling interval).

## 📂 Структура решения

- `Backend`: REST API сервер, обрабатывающий запросы терминала и управляющий Mock-сервисами.
- `Core`: Ядро системы с бизнес-логикой, контрактами сервисов и правилами валидации.
- `Terminal`: Десктопное приложение для пользователя/инженера.
- `Unit`: Проект с автоматическими тестами.
